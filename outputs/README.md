# Outputs — `prediction_result.csv`

This folder contains the final prediction file generated by the trained models on the **holdout dataset** (February – March 2025, 38,859 sessions).

---

## File

| File | Rows | Columns |
|---|---|---|
| `prediction_result.csv` | 38,859 | 6 |

---

## Column Definitions

| Column | Type | Description |
|---|---|---|
| `user_id` | Integer | Unique identifier for the website user (matches `user_id` in `website_sessions`) |
| `pred_score` | Binary (0/1) | **Binary purchase prediction.** `1` = the model predicts this user will place an order; `0` = no order predicted |
| `pred_multi_score_1` | Binary (0/1) | **Product 1 (CorePack) prediction.** `1` = predicted to purchase CorePack; `0` = not predicted |
| `pred_multi_score_2` | Binary (0/1) | **Product 2 (TechFortress) prediction.** `1` = predicted to purchase TechFortress; `0` = not predicted |
| `pred_multi_score_3` | Binary (0/1) | **Product 3 (AirLite) prediction.** `1` = predicted to purchase AirLite; `0` = not predicted |
| `pred_multi_score_4` | Binary (0/1) | **Product 4 (EcoShell) prediction.** `1` = predicted to purchase EcoShell; `0` = not predicted |

> **Note:** `pred_multi_score_1` through `pred_multi_score_4` are only populated for users where `pred_score = 1` (i.e., users predicted to make a purchase). For non-buyers, all multi-class scores are `0`.

---

## How Predictions Were Generated

### Binary Predictions (`pred_score`)

The binary column was generated using a **GBT (Gradient Boosted Trees) Classifier**, selected as the best-performing model across five candidates:

- Trained on a chronological split of historical session data (March 2022 – March 2024)
- Validated and tuned on the validation period (March 2024 – August 2024)
- Optimal hyperparameters: `maxDepth=5`, `maxIter=20`, `stepSize=0.1`
- Final test AUC-PR: **0.1574** | Test AUC-ROC: **0.7774** | Test Accuracy: **92.26%**

The holdout dataset underwent the same feature engineering pipeline (30+ features across temporal, behavioural, product-level, and historical dimensions) before predictions were generated.

### Multi-Class Predictions (`pred_multi_score_1` to `pred_multi_score_4`)

The product-level columns were generated using a **Random Forest Classifier** (multi-class):

- Trained on sessions where a purchase was confirmed, predicting the purchased `product_id`
- Validation Accuracy: **98.02%** | Validation F1-Score: **0.9720**
- Products are mapped to their IDs as defined in the `products` table (CorePack=1, TechFortress=2, AirLite=3, EcoShell=4)

## Usage

Load the file in Python:

```python
import pandas as pd

df = pd.read_csv("prediction_result.csv")
print(df.head())
print(f"Total users: {len(df)}")
print(f"Predicted buyers: {df['pred_score'].sum()} ({df['pred_score'].mean()*100:.1f}%)")
```

Or in PySpark:

```python
df = spark.read.csv("prediction_result.csv", header=True, inferSchema=True)
df.show(5)
```
